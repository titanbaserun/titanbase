# Security Policy

## Reporting vulnerabilities

If you discover a security vulnerability in Titanbase, please report it responsibly.

**Do NOT open a public GitHub issue for security problems.**

Instead, email: **security@titanbase.run**

Include:
- Description of the vulnerability
- Steps to reproduce
- Impact assessment (if possible)

We will acknowledge receipt within 48 hours and provide a timeline for a fix.

## Current status

Titanbase is in early development (v0.x). The open-source editor runs entirely client-side with no backend, network calls, or data persistence beyond the user's own browser localStorage and file system.

There is currently no server, no authentication, and no cloud component in this repository.

## Scope

This policy covers the code in this repository (`titanbaserun/titanbase`). Cloud services (when launched) will have their own security policy.
